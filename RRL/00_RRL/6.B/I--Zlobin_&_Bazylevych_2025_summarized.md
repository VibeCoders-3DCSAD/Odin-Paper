## TL;DR

XGBoost outperforms all models in credit scoring, random forest achieves **99.6%** fraud detection accuracy, and LSTM reaches 93% stock prediction accuracy, but model interpretability remains the key barrier to adoption in finance.

## Problem and Motivation

Traditional statistical methods struggle to process the large, complex datasets generated by modern financial markets. Financial institutions need more accurate predictive models for credit risk, fraud detection, and trading to improve decision-making and reduce losses. Prior reviews lacked a systematic, side‑by‑side comparison of ML and DL methods across both classification and regression tasks with explicit performance metrics and trade‑off analysis.

## Approach

- 41 peer‑reviewed papers analyzed from 2007–2025 covering ML/DL in finance.
- Classification tasks: credit scoring, fraud detection, customer segmentation.
- Regression tasks: stock price prediction, volatility forecasting, option pricing.
- Models compared: decision trees, SVM, logistic regression, random forest, XGBoost, DBN, CNN, LSTM, GCN, hybrid (CNN‑AdaBoost, BS‑ANN, GRU‑CA).
- Evaluation metrics: accuracy, AUC, RMSE, F1‑score, precision, recall, Matthews correlation coefficient.
- Challenges identified: model interpretability (“black box”), data imbalance, noise, concept drift.

## Findings

1. Random forest with ensemble feature selection achieved **99.6% accuracy and 100% precision** in credit card fraud detection (F1‑score 99.6%).
2. XGBoost consistently outperformed all deep learning models across 10 credit scoring datasets (highest AUC, lowest Brier score, and highest expected maximum profit).
3. LSTM networks predicted Vietnamese stock price trends with **93% accuracy**, outperforming linear regression models.
4. GRU‑CA hybrid model reduced anomaly detection RMSE on S&P 500 data from 13.28 to **9.74** (≈27% improvement) by adding a contextual attention module.
5. GCN achieved 94.5% fraud detection accuracy vs. CNN’s 93% on insider threat data, improving recall by 10 percentage points.

- CNN‑AdaBoost hybrid reached 96.35% accuracy for electricity theft detection, beating standalone CNN and AdaBoost.
- BS‑ANN hybrid option pricing model reduced pricing errors and achieved the lowest standard deviation among all tested models.
- AI personalization in fintech increased customer retention by 15% and engagement by 27% via collaborative filtering.

## Definitions

| Term / Acronym | Plain-English Definition |
| -------------- | ------------------------ |
| ML | Machine learning – algorithms that learn patterns from data without explicit programming. |
| DL | Deep learning – multi‑layer neural networks that automatically extract hierarchical features. |
| AUC | Area Under the ROC Curve – metric ranking a model’s ability to separate positive from negative classes (0.5 = random, 1.0 = perfect). |
| RMSE | Root Mean Square Error – typical prediction error magnitude; lower is better. |
| LSTM | Long Short‑Term Memory [think: memory cell with forget/input/output gates for sequences] – recurrent neural network that learns long‑term dependencies in time series. |
| CNN | Convolutional Neural Network – extracts spatial/local patterns; used here on tabular data converted to images. |
| GCN | Graph Convolutional Network – learns from relationship graphs (user‑to‑user connections) for fraud detection. |
| RF | Random Forest – ensemble of decision trees; robust to overfitting, handles imbalanced data well. |
| XGBoost | Extreme Gradient Boosting – boosted tree ensemble; often best for tabular data with missing values. |
| Concept drift | When the statistical properties of a target variable change over time (e.g., spending patterns after job loss). |
| Federated learning | Training models across decentralized data sources without sharing raw data – preserves privacy. |
